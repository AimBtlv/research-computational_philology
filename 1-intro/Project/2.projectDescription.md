**Шаг 3 — Понимание Структуры CoNLL-U**
Прежде чем сравнивать — нужно понять что смотреть. Каждая строка в CoNLL-U выглядит так:

```
ID  FORM LEMMA UPOS XPOS  FEATS HEAD DEPREL DEPS  MISC 
```
**Нас интересуют два столбца:**
**HEAD** — номер токена-головы (от кого зависит)
**DEPREL** — тип синтаксического отношения (nsubj, obj, advmod и т.д.)
**Основные DEPREL которые встретятся:**

| DEPREL   | Значение                                                                                                                                                                                                 | Пример               |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------- |
| `nsubj`  | подлежащее                                                                                                                                                                                               | _Patricius_ peccator |
| `root`   | корень предложения (главный глагол)                                                                                                                                                                      | _dixit_              |
| `obj`    | прямое дополнение                                                                                                                                                                                        | ignorabam _Deum_     |
| `obl`    | косвенное дополнение / обстоятельство                                                                                                                                                                    | _in captivitate_     |
| `amod`   | согласованное определение                                                                                                                                                                                | _verum_ Deum         |
| `advmod` | обстоятельство                                                                                                                                                                                           | _tunc_               |
| `conj`   | сочинительная связь                                                                                                                                                                                      | et _dispersit_       |
| `cc`     | союз при conj                                                                                                                                                                                            | _et_                 |
| `xcomp`  | зависимая предикация                                                                                                                                                                                     | _fuisse_             |
| ccomp    | - _Она сказала, что **[он ушёл]**_ — `ccomp`: "он" ≠ "она"<br>- _Caesar dicit **[milites venisse]**_ — milites (acc) = subj инфинитива, ≠ Caesar<br>- _Я знаю, что **[ты права]**_ — `ccomp`: "ты" ≠ "я" |                      |
**Главный Тест для Различения ccomp и xcomp**
Единственный надёжный вопрос: **у зависимой клаузы есть свой субъект?**

|Ситуация|Отношение|
|---|---|
|_Она сказала, что **он** ушёл_ — субъекты разные|`ccomp`|
|_Она хочет уйти_ — субъект один|`xcomp`|
|_Он видит, что **птицы** летят_ — разные|`ccomp`|
|_Он начал **читать**_ — один субъект|`xcomp`|
|_Caesar dicit **milites** venisse_ — acc = свой subj|`ccomp`|
|_Caesar vult **vincere**_ — тот же Caesar|`xcomp`|
Полный список: [https://universaldependencies.org/u/dep/](https://universaldependencies.org/u/dep/)

**4.2 Для каждой пары предложений сравни DEPREL**
Ищи **расхождения в синтаксических ролях** для семантически эквивалентных слов.
**Пример — первое предложение:**
Латынь: _Ego Patricius peccator rusticissimus..._ Английский: _My name is Patrick...

|Смысл|Латынь|Ожидаемый DEPREL|Английский|DEPREL|
|---|---|---|---|---|
|субъект|_Ego_|`nsubj`|_I_|`nsubj`|
|имя|_Patricius_|`appos` или `nsubj`|_Patrick_|`attr`/`nsubj`|
|характеристика|_peccator_|`appos`|_sinner_|`nsubj`|

Если UDPipe присвоил латинскому _peccator_ тег `nsubj`, а в английском это `appos` — **нужно исправить**.

**4.3 Исправление в CoNLL-U**
Открой CoNLL-U файл в текстовом редакторе (или используй скрипт из Шага 5).
**Что менять:** 
-Столбец 7 (HEAD) — если структура дерева неверная
-Столбец 8 (DEPREL) — если тип отношения неверный
```
# Было (неверно):
5   peccator  peccator  NOUN  _  Case=Nom  3  nsubj  _  _

# Стало (верно):
5   peccator  peccator  NOUN  _  Case=Nom  3  appos  _  _
```
